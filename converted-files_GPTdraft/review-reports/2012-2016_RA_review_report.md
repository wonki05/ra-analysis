# LEET 추리논증 문제지 Markdown 변환 검수 결과 요약

## 검수 대상

- **연도**: 2012 ~ 2016
- **과목**: 추리논증
- **약어**: RA
- **원문 비교 대상**: `[연도]_LEET_추리논증_문제지(홀수형).pdf`
- **검수 대상**: `[연도]_RA_improved_output.zip`

## 총괄 요약

- 수정이 반드시 필요한 실질 오류는 **2012년 24번 문항의 도식 이미지 누락**입니다.
- 그 외 연도에서는 문항 수, 문항 번호, 본문·선택지의 주요 텍스트, 표·수식·이미지 참조에서 명백한 누락은 발견되지 않았습니다.
- 다만 일부 표와 선택지 배열은 원문 PDF의 시각적 배치와 다르게 Markdown/HTML 표 또는 선형 목록으로 변환되어 있어, GitHub 렌더링 화면에서 최종 확인을 권장합니다.

## 2012년 RA

- **문항 수 및 문항 번호**
  - Markdown 파일에 **1번부터 35번까지 전 문항이 존재**합니다.
  - 문항 번호의 누락이나 중복은 발견되지 않았습니다.

- **본문·선택지 텍스트**
  - 전체 문항의 본문과 선택지는 원문 PDF와 대체로 일치합니다.
  - 줄바꿈, 띄어쓰기, 선택지 배열, `<보기>`의 Markdown 제목화 등 형식 차이는 있으나, 의미 있는 글자·단어 누락으로 보이는 부분은 24번 이미지 누락 외에는 발견되지 않았습니다.

- **이미지·도식**
  - 12번의 `<그림 1>`, `<그림 2>` 이미지는 압축파일 내 이미지와 Markdown 참조가 존재하며 원문과 대응됩니다.
  - 33번, 34번 그래프 이미지도 압축파일에 포함되어 있고 Markdown에서 참조됩니다.
  - **오류: 24번 문항의 도식 이미지가 누락되었습니다.**
    - 원문 PDF 24번에는 `<설명>` 아래에 원통, 금속판 `S1`, `S2`, 접착제, 실 등이 표시된 실험 장치 도식이 있습니다.
    - 검수대상 Markdown에는 이 도식 이미지 참조가 없습니다.
    - 압축파일 안에도 `2012_RA_images_24_*.png`에 해당하는 이미지가 없습니다.
    - 이 도식은 문항 풀이에 필요한 정보이므로 **반드시 이미지 파일을 추가하고 24번 본문 중 `<인과 이론>` 앞에 삽입해야 합니다.**

- **표·수식**
  - 17번 표, 31번의 복수 표는 원문 PDF와 비교했을 때 행·열 항목과 수치가 보존되어 있습니다.
  - 5번, 12번 등 일부 선택지 표는 원문의 가로 배열이 Markdown 표 또는 선형 선택지 구조로 변환되어 있으나, 내용 자체의 누락은 발견되지 않았습니다.

- **강조표시**
  - 11번의 밑줄 발화 `㉠`～`㉣`은 `<u>...</u>` 형식으로 보존되어 있습니다.
  - 원문상 의미 있는 강조표시가 Markdown에서 빠진 사례는 24번 이미지 누락 외에는 발견되지 않았습니다.

## 사용자가 직접 확인했으면 하는 부분

- **최우선 수정 필요: 2012년 24번 문항 도식 누락**
  - 원문 PDF의 실험 장치 도식이 Markdown과 압축파일 이미지 목록에서 모두 빠져 있습니다.
  - 이 도식은 문항 조건의 핵심 정보이므로, 단순 형식 문제가 아니라 **문항 정보 누락**입니다.
  - `2012_RA_images_24_1.png`와 같은 파일명으로 이미지를 추가하고, 24번 문항의 `<설명>` 부분 중 장치 설명 직후, `<인과 이론>` 앞에 삽입하는 방식이 적절합니다.

- **2016년 20번의 `**` 표기**
  - 원문에서 주석 표지처럼 보이는 `**`가 Markdown에 그대로 들어가 있습니다.
  - 일반적으로는 공백 때문에 굵게 처리되지 않을 가능성이 높지만, GitHub 렌더링 안정성을 높이려면 `\*\*`처럼 이스케이프할지 확인하는 것이 좋습니다.

- **표·선택지 배열의 렌더링 확인**
  - 2012년 5번·12번, 2015년 5번, 2012년 31번, 2013년 27번, 2015년 31번, 2014년 35번처럼 원문에서 표나 행렬 형태였던 부분은 내용은 보존되어 있으나 Markdown 렌더링에서 시각적 배열이 달라질 수 있습니다.
  - AI 분석 목적이라면 현재 구조도 대체로 문제없지만, GitHub에서 사람이 읽는 용도까지 고려한다면 렌더링 화면에서 행·열 정렬을 한 번 더 확인하는 것이 안전합니다.
