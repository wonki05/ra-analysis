# 2023_RA 공식 해설 Markdown 가공 작업 보고

## 1. 판독이 어려웠던 부분

원본 PDF는 텍스트 레이어가 없는 스캔 이미지 기반 문서였으므로, 300dpi 렌더링 이미지를 기준으로 OCR 결과와 원본 페이지 이미지를 대조하였다. 특히 라틴 문자 변수(A, B, C, D, R, W 등), 선택지 번호, 원문 속 따옴표, 수식 기호, 첨자 표기에서 OCR 혼동이 발생하기 쉬웠다.

검수 중 확인된 OCR 혼동 중 의미가 바뀔 수 있는 항목은 원본 이미지와 대조하여 수정하였다. 예를 들어 `옮/옳`, `감/갑`, `SNS`, `제1조/제2조`, `B등급/C등급`, `범죄/DNA` 등은 고해상도 페이지 이미지와 대조하여 보정하였다.

## 2. Markdown으로 옮기면서 판단이 필요했던 부분

문항별 원문의 `문항 성격` 표는 기존 지침에 맞추어 `문항유형 / 내용영역` 형식으로 옮겼다. 원문의 `문제 풀이`, `정답 해설`, `오답 해설`, `<보기> 해설` 표식은 원문 순서를 유지하되 지정 구조에 따라 `### 제시문 해설`, `### <보기> 해설`, `### 선택지별 해설` 아래에 배치하였다.

32번과 34번은 공간 배치가 해설의 핵심이므로, 텍스트로 무리하게 대체하지 않고 해당 도식만 분리 crop하였다. 도식 앞뒤의 원문 설명 문장은 Markdown 본문으로 전사하고, 도식은 원문에서 등장하는 위치에 이미지 링크로 삽입하였다.

## 4. 표 처리 방식

단순 표는 Markdown 표로 옮겼다. 셀 병합이 있는 표는 Markdown에서 병합을 직접 표현할 수 없으므로, 같은 의미가 유지되도록 열 제목을 반복하거나 행 단위 정보를 풀어서 배치하였다. 33번의 성과급 표는 원문 표의 행과 열 순서를 유지하면서 Markdown 표로 재구성하였다. 공간 관계 자체가 핵심인 32번 상자 배열과 34번 주차 배열은 표로 바꾸지 않고 이미지로 처리하였다.

## 5. 원문과 달라질 수밖에 없었던 표현 방식 및 이유

원문의 선택지 번호 `①`~`⑤`는 작업 지침에 따라 Markdown에서는 `(1)`~`(5)`로 통일하였다. 페이지 번호, 머리말, 꼬리말, 반복되는 책 제목, 장식용 구분선은 제외하였다. 원문 표의 셀 병합과 회색 배경은 GitHub Markdown에서 그대로 재현하기 어려우므로, 의미와 행ㆍ열 순서를 보존하는 방식으로 단순화하였다. 이미지 링크는 같은 폴더에 PNG가 있다는 전제로 `./파일명.png` 형식으로 작성하였다.
